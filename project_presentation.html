system architecture

Three layers,
one serial wire

Robot Framework runs .robot test suites on the host PC. It talks to the STM32 testbench over a USB-TTL UART dongle using Zephyr's built-in shell. The testbench then drives or measures the DUT through GPIO, SPI, UART, ADC, and PWM — all selectable at runtime, no firmware rebuild needed.

The shell protocol is human-readable ASCII. Every command is tb <peripheral> <action> <args>, and every response is either a value or ERROR: .... That one convention handles retry logic, timeout detection, and Robot Framework failure propagation cleanly.

implemented features

What the testbench can do

[GPIO]

GPIO control

Drive any pin high or low and read back the digital level in a single shell round-trip. Configures direction automatically on each call.

tb gpio set PA5 1
[ADC]

ADC measurement

12-bit ADC read on any mapped channel, returned in millivolts. Internally uses Zephyr's adc_read() driver with the configured reference voltage.

tb adc read PA0
[SPI]

SPI transfer

Full-duplex SPI transaction of up to 64 bytes. TX bytes are hex-encoded arguments; RX bytes are returned in the same format.

tb spi xfer 0 DE AD
[UART]

DUT UART bridge

Send strings to and receive strings from the DUT over a second UART (USART2), independent of the shell UART on USART1. Ring-buffer backed RX.

tb uart send uart2 hello
[PWM]

PWM output & capture

Generate a PWM signal at any frequency and duty cycle. Capture an incoming PWM signal (frequency and duty) using timer input-capture mode.

tb pwm set PA8 1000 50
[SIM]

Motor plant simulation

Simulink-generated DC motor model (ODE3 solver, 2 states) runs on a Zephyr periodic timer. Reads voltage command from ADC or PWM, streams speed and current back to the host.

tb motorsim start adc PA0

Simulink → C code generation

The DC motor plant model is built in Simulink and exported to self-contained C with Embedded Coder targeting ARM Cortex-M — producing the generated dc_motor.c that compiles directly into the firmware build.

[PLOT]

Simulation plotting script

Standalone Python script that streams the motor-sim CSV output and renders speed and current live on a Qt plot — runs on its own, no Robot Framework required.

python motorsim_monitor.py

stage 10 — dc motor simulation

Simulink plant running
on bare metal

The DC motor model was built in Simulink using plain transfer-function blocks (no Simscape), then exported as self-contained C code via Embedded Coder targeting ARM Cortex-M. It runs inside a Zephyr workqueue thread stepped every 200 ms — no RTOS modifications, no external runtime library.

Plant equations

La · dI/dt = V − Ra·I − Ke·ω ← armature current
J · dω/dt = Kt·I − b·ω ← rotor speed
Parameter definitions
R (resistance)
2.0 Ω — armature winding
L (inductance)
0.05 H — armature inductance
J (inertia)
0.0005 kg·m²
K (Ke = Kt)
0.1 N·m/A
b (damping)
0.0003 N·m·s
solver
ODE3 (Bogacki-Shampine)
step size
200 ms (configurable)

Firmware integration

tb_motorsim.c 
/* Called by Zephyr workqueue, NOT in ISR context */
static void motorsim_work_handler(struct k_work *work)
{
  /* 1. Read DUT command from ADC or PWM capture */
  dc_motor_math_U.Voltage = read_input_voltage();

  /* 2. Step the Simulink-generated ODE3 solver */
  dc_motor_math_step();
  step_count++;

  /* 3. Stream result directly to shell UART */
  if (stream_enabled) {
    real_T speed   = dc_motor_math_Y.Speed;
    real_T current = dc_motor_math_X.Integrator_CSTATE;
    snprintf(line, sizeof(line),
      "MSIM,%u,%d.%03d,%d.%03d,%d.%03d\n",
      step_count * MOTORSIM_STEP_MS,
      /* voltage, speed, current decomposed */ …);
    shell_uart_write_line(line, strlen(line));
  }
}

Robot Framework
keyword library

ZephyrShellLib translates every shell command into a Robot Framework keyword. It handles prompt detection, timeout retries, thread locking for parallel runners, and raises clean failures on ERROR: responses — so test code stays readable.

Python transport core

ZephyrShellLib/transport.py (simplified) 
class ShellTransport:
    def run(self, cmd: str, timeout: float = None) -> str:
        t = timeout or self._default_timeout
        self._ser.reset_input_buffer()
        self._ser.write((cmd + "\r\n").encode())
        raw = self._drain(timeout=t)   # read until prompt
        return self._extract(raw, cmd)  # strip echo + prompt

    def _drain(self, timeout):
        buf = b""
        deadline = time.monotonic() + timeout
        while time.monotonic() < deadline:
            buf += self._ser.read(256)
            if self.prompt in buf:
                return buf  # prompt = "blackpill:~$ "
        raise TimeoutError(f"No prompt. Buffer: {buf!r}")
